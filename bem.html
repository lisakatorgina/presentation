БЭМ

— блок, элемент, модификатор
Методология созданная в Яндексе в ходе оптимизации процесса разработки интерфейсов.

  • Простая поддержка структуры кода при росте проекта.
  • Повторное использование существующего кода.
  • Точечные изменения с минимальными затратами: обновление дизайна, добавление функциональных элементов и т.д.
Блок — логически и функционально независимый компонент страницы.
  • свободное перемещение
  • повторное использование
Элемент — Составная часть блока, которая не может использоваться в отрыве от него.
Модификатор — БЭМ-сущность, определяющая внешний вид, состояние и поведение блока или элемента.

Правила именования
Основная идея — сделать имена CSS-селекторов максимально информативными и понятными.

block-name__elem-name

			<div class="menu">
				<span class="menu__item"></span>
			</div>

булевые модификаторы — block-name_mod-name
ключ-значение — block-name_mod-name_mod-val

			<div class="menu menu_hidden">...</div>
			<div class="menu menu_theme_green">...</div>

модификатор элемента — block-name__elem-name_mod-name

			<div class="menu">
				<span class="menu__item menu__item_active menu__item_view_big">
			</div>

Организация файловой системы.
Так же как интерфейс делится на независимые компоненты — блоки, реализация блоков делится на независимые части — файлы.

			blocks/
				menu/
					menu.css
					menu.js
				button/
					button.css
					button.js
					i/
						button.png
Миксы 
			<ul class="menu header__item">
				<li class="menu__item">
					<a href="" class="menu__link link">
				</li>
			</ul>
Каскады 
			.nav__item_active .nav__link {
			    color: red;
			}
